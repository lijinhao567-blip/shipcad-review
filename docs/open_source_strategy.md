# 开源项目策略

## 项目定位

本项目定位为 AGPL-3.0 开源智能审图平台，目标是让船舶 CAD 图纸审查流程具备可复现、可审计、可扩展的工程基础。

核心方向：

- DXF/DWG 解析：DXF 使用 ezdxf，DWG 使用 LibreDWG 转换链路。
- 规则审查：先用可解释规则保证稳定性。
- 符号识别：使用 YOLOv8 作为独立 Vision Worker。
- AI 辅助：用于摘要、解释、报告和后续知识图谱推理。

## 许可证选择

主项目采用 GNU AGPL-3.0。原因：

- YOLOv8 默认使用 AGPL-3.0。
- LibreDWG 使用 GPLv3+。
- 项目预期部署为 Web 服务，AGPL 能覆盖网络服务形态下的源码开放要求。

## GitHub 上架边界

可以进入仓库：

- 源码、接口文档、架构文档、规则样例。
- 自造或公开许可允许的 DXF/DWG 样例。
- 数据集结构说明和标注格式说明。

不进入仓库：

- 真实船舶图纸。
- 用户数据、数据库文件、日志文件。
- 大模型权重和 YOLO 训练输出。
- 本机工具链、虚拟环境、商业 SDK。

## 商业化方式

AGPL 开源并不排斥商业化。更适合的方式是：

- 私有化部署服务。
- 行业规则库建设。
- 标注数据和模型训练服务。
- 企业集成、运维、培训和定制开发。
